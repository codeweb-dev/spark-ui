The Interactive Pets component renders a small bounded playground with a cat, a dog, and a bird. Each pet can be dragged anywhere inside the playground (mouse, touch, or keyboard) and fed with its matching food — fish for the cat, a bone for the dog, seeds for the bird. Feeding flies the food to the pet, bounces it, and pops a short reaction bubble.

```tsx

  return <InteractivePets />;
}
```

### Custom Messages

Override any pet's name, starting position, or messages. Pets you omit are left out entirely.

```tsx
<InteractivePets
  pets={[
    { id: "cat", name: "Nimbus", fedMessage: "an acceptable offering." },
    { id: "dog", name: "Waffles", fedMessage: "BEST. DAY. EVER." },
    { id: "bird", name: "Pixel", fedMessage: "chirp of approval" },
  ]}
/>
```

### Event Callbacks

`onPetMove` fires after a drag ends or a keyboard move, with the pet's position in pixels relative to the playground. `onPetFeed` fires when a food button is pressed.

```tsx
<InteractivePets
  onPetMove={(pet, position) =>
    console.log(`${pet} moved to`, position)
  }
  onPetFeed={(pet) => console.log(`${pet} was fed`)}
/>
```

## Accessibility

- Each pet is focusable and exposes an accessible label (e.g. "Mochi the cat. Drag to move, or use arrow keys.").
- Focused pets move with the arrow keys (8px per press, 24px with <kbd>Shift</kbd>) and stay inside the playground.
- Food controls are native buttons with descriptive `aria-label`s.
- Feeding reactions are announced through a polite `aria-live` region.
- A visible focus ring is shown on pets and food buttons.

## Reduced Motion

When `prefers-reduced-motion` is set, idle animations (tail sways, bobbing) are disabled, the food travel animation is skipped, and reaction bubbles appear and disappear without transitions — feeding still shows the reaction bubble and fires callbacks, so nothing is lost besides the motion.
